GlowCRM — One app to run a tanning business.

Built for tanning owners who are tired of juggling eight different tools. Book clients, send reminders, record formulas, track packages, and collect payment — all in one place.

Client Bronz Bliss, Cedar City UT
Category Vertical SaaS / CRM
Status MVP Built — Going Live
Stack React · Express · SQLite · Tailwind

The Problem

Eight tools to run one business.

The first client — a spray tan studio owner — was stitching together a different app for every part of her workflow. No central record, no automation, no visibility into what was actually driving her business.

After GlowCRM

GlowCRM

Booking, client profiles, session recording, SMS reminders, package tracking, payments, and analytics — all in one app built specifically for the way a tanning business actually operates.

What We Built

Built around the tanning workflow — not generic CRM logic.

Every feature maps to a real decision a tanning owner makes every day. No bloat, no features that sound good on a demo and never get used.

Appointment Calendar

Day-by-day view with time-sorted appointment cards. Create, check in, or cancel directly from the timeline.

Live

Client Profiles

Fitzpatrick skin type, allergies, preferred formula, session history, and active packages stored per client.

Live

Session Check-In

Formula and solution recording, shade, rinse time, aftercare notes, and checkout — all in one tanning-specific flow.

Live

SMS Reminders

Automated appointment confirmations, prep instructions before the visit, and rinse reminders after the session via Twilio.

Phase 2

Public Booking Link

A shareable URL so clients self-book without calling or texting. Removes the owner from the scheduling loop entirely.

Phase 2

Packages & Payments

Create session-bundle plans (e.g., Glow 5-Pack), auto-deduct sessions on check-in, and process real payments via Stripe.

Phase 1 → 2

Revenue & Analytics

Dashboard KPIs, revenue by period, top services, client retention rate, and appointment volume — updated in real time.

Phase 1 → 3

Bridal & Party Bookings

Self-serve group event booking replaces DM-based bridal inquiries — the highest-value service gets a real checkout flow.

Phase 3

AI Rebooking Agent

Uses session history to flag ideal rebooking windows, trigger follow-up texts, and surface churn-risk clients before they go quiet.

Phase 3

Roadmap

From sister's business to vertical SaaS.

Validating with a real operator first, then opening to other tanning business owners. Every phase is designed to create pull before adding scope.

Tech Stack
